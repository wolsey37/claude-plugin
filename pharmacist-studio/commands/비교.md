입력: $ARGUMENTS

## 실행 규칙

- 제품명 1개만 입력 → /성분 스킬로 자동 전환
- "알아서 해줘" 입력 → 올리브영 베스트 기준 자동 선정 (Chrome MCP로 수집)
- 제품명 2개 이상 → 비교표 출력

## 비교 항목
각 제품에 대해 웹 검색 또는 Chrome MCP로 전성분 수집 후:
1. 핵심 기능 성분 (유효 농도 추정)
2. 1% 기준선 위치
3. 콘텐츠 후킹 포인트
4. 약사 종합 점수 (5점 만점)

## 💾 옵시디언 저장 (정돈된 노트)
출력과 **동시에** 결과를 전용 보관함에 보기 좋은 노트로 저장한다.
- **보관함:** 작업 폴더(현재 작업 디렉토리) 아래 `obsidian/` 폴더
  - **보관함이 없으면 자동 생성한다.** 단순 폴더가 아니라 Obsidian이 보관함으로 인식하도록 `obsidian/.obsidian/` 폴더와 내용이 `{"showUnsupportedFiles": true}`인 `obsidian/.obsidian/app.json`도 함께 만든다(HTML 첨부도 탐색기·링크에서 보이도록).
- **저장 폴더:** `<보관함>/비교/` · **파일명:** `<주제·키워드>_<YYYYMMDD>.md` (공백·특수문자는 `_`)

`Write`로 아래 구조의 노트를 만든다 — Obsidian 속성(properties)·태그·콜아웃으로 깔끔하게:

```markdown
---
유형: 비교
주제: "<주제·키워드>"
생성일: <YYYY-MM-DD>
제품수: <N>
추천1위: "<제품명>"
tags:
  - 약사스튜디오
  - 비교
  - <카테고리>        # 예: 선크림
  - <핵심성분>        # 비교 제품들의 공통·핵심 성분
---

> [!note] 최종 추천
> <1위 제품과 이유 한 줄>

<결과 본문 — 위 출력 포맷 그대로>
```

- **연결용 태그(필수):** 태그의 `<카테고리>`·`<핵심성분>`·`<피부고민>`·`<트렌드키워드>`·`<콘텐츠앵글>` 자리표시자를 **실제 값으로 바꾼다**(해당 없으면 그 줄 삭제). 범용 태그(약사스튜디오·유형)뿐 아니라 **카테고리·성분 태그**를 넣어야 같은 성분/카테고리를 쓰는 다른 제품과 그래프·태그로 자동 연결된다. (예: `선크림`, `무기자차`, `나이아신아마이드`)
- **개념 허브 연결:** 핵심성분·카테고리·효능마다 `_허브/<개념>.md`(없으면 생성)의 `## 관련 제품`에 이 노트 링크를 추가해 위키링크로 연결한다.
- 저장 실패해도 출력은 정상 진행한다.
- **마무리 안내(필수):** 출력 맨 끝에 아래 2가지를 항상 알려준다 — Obsidian 보관함 자동 등록은 하지 않으므로, 사용자가 직접 "폴더를 보관함으로 열기" 할 수 있게 경로만 제공한다.
  - 📁 **보관함 경로:** `<작업 폴더>/obsidian` (절대경로로)
  - 📄 **저장된 노트:** 방금 저장한 파일의 전체 경로
