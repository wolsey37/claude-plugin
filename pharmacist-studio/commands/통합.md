주제: $ARGUMENTS

## 실행 순서

아래 5단계를 순서대로 자동 실행 후 **하나의 통합 리포트**로 출력한다.

> **각 단계는 통합 리포트에 합치는 동시에, 해당 폴더에 개별 노트로도 저장한다.** (`/통합` 한 번에 5개 폴더가 모두 채워지도록)

### ① /수집
Chrome MCP로 올리브영 + 쓰레드에서 [주제] 관련 데이터 수집.
- **개별 저장:** 결과를 `수집/[주제]_[YYYYMMDD].md`에 `/수집` 스킬의 형식(프런트매터·태그·콜아웃)대로 저장.

### ② /성분
수집된 베스트 상품 1~2개의 전성분 분석 (웹 검색으로 전성분 확보 후 식약처 기준 평가).
- **개별 저장:** 결과를 `성분/[주제 또는 제품명]_[YYYYMMDD].md`에 `/성분` 스킬 형식대로 저장.

### ③ /비교
수집된 상위 제품 2~3개 비교표 출력.
- **개별 저장:** 결과를 `비교/[주제]_[YYYYMMDD].md`에 `/비교` 스킬 형식대로 저장.

### ④ 콘텐츠 기획 방향 도출
수집·성분·비교 결과를 종합해 약사 크리에이터 콘텐츠 기획안 작성.

### ⑤ /각본 (각본 자동 생성)
④에서 정한 **1순위 콘텐츠**로 `/각본` 스킬을 실행해 30~60초 쇼츠 각본(후킹 후보 3개·본론·CTA·촬영 암기 키워드)을 작성한다.
- 통합 리포트 본문에 "🎬 영상 각본" 섹션으로 포함하고,
- **별도 마크다운 노트로 `각본/[주제]_[YYYYMMDD].md`에도 저장**한다(`/각본` 스킬의 저장 형식·프런트매터 그대로).

> **결과적으로 `/통합 [주제]`는 `수집/ · 성분/ · 비교/ · 통합/ · 각본/` 5개 폴더 + `_첨부/` HTML까지 한 번에 저장한다.** 각 개별 노트의 프런트매터에 `상위리포트: "[[통합/[주제]_[YYYYMMDD]]]"`를 넣어 통합 노트와 상호 연결한다.

## 최종 출력: HTML 리포트 파일 생성

4단계 결과를 종합해 **하나의 완성된 HTML 파일**을 `Write` 도구로 생성한다.

> **보관함 자동 생성:** 작업 폴더 아래 `obsidian/` 보관함이 없으면 자동 생성한다. 단순 폴더가 아니라 Obsidian이 보관함으로 인식하도록 `obsidian/.obsidian/` 폴더와 내용이 `{}`인 빈 `obsidian/.obsidian/app.json`도 함께 만든다.

- **파일 경로:** 옵시디언 보관함 `_첨부` 폴더에 저장
  → `obsidian/_첨부/통합리포트_[주제]_[YYYYMMDD].html`
- 마크다운이 아니라 **그대로 브라우저에서 열리는 완결된 HTML 문서**로 만든다.
- 생성 후 파일 경로를 사용자에게 안내하고, 필요 시 Chrome MCP `navigate`로 `file://` 경로를 열어 미리보기한다.
- 표·점수·키워드는 아래 템플릿의 클래스를 그대로 활용해 시각화한다.

### 옵시디언 요약 노트도 함께 저장 (정돈된 노트)
HTML과 별도로, 보관함에서 바로 읽을 수 있는 **마크다운 요약 노트**를 아래 구조로 저장한다.
- **경로:** `obsidian/통합/[주제]_[YYYYMMDD].md`

```markdown
---
유형: 통합
주제: "<주제>"
생성일: <YYYY-MM-DD>
추천1위: "<제품명>"
리포트: "[[_첨부/통합리포트_<주제>_<YYYYMMDD>.html]]"
tags:
  - 약사스튜디오
  - 통합
---

> [!abstract] 한눈에 보기
> <시장·성분·비교를 종합한 핵심 결론 한 줄>

## 📊 4단계 핵심 요약
- **시장(수집):** ...
- **성분:** ...
- **비교:** ...

## 🎯 콘텐츠 기획 방향
- ...

> [!tip] 전체 리포트는 [[_첨부/통합리포트_<주제>_<YYYYMMDD>.html]] 에서 확인
```

- `리포트` 속성과 본문 링크로 HTML 리포트가 노트에서 바로 열리게 한다.
- 저장 실패해도 결과 출력은 정상 진행한다.
